Vestix — Laravel Forge setup checklist
========================================

Server (Forge → Create Server)
  [ ] PHP 8.3
  [ ] MySQL 8
  [ ] Node.js enabled
  [ ] UFW / fail2ban defaults

Site (Forge → Create Site)
  [ ] Domain: vestix.io
  [ ] Web directory: /public
  [ ] Database (modal "Add database" tijdens site-aanmaak):
        Name:     vestix
        User:     forge           (Forge default — laat leeg of vul "forge" in)
        Password: [Generate password] — kopieer direct, je ziet hem daarna niet meer
  [ ] SSL: Let's Encrypt (DNS A-record voor vestix.io → server IP)
  [ ] Git: git@github.com:301-ads/vestix.git, branch main
  [ ] Deploy on push: optional

Environment (Forge → Site → Environment)
  [ ] Copy forge/production.env.example as starting point
  [ ] Set APP_KEY (php artisan key:generate --show)
  [ ] Set APP_URL=https://vestix.io
  [ ] Database (.env moet matchen met wat je in Forge aanmaakte):
        DB_CONNECTION=mysql
        DB_HOST=127.0.0.1
        DB_PORT=3306
        DB_DATABASE=vestix
        DB_USERNAME=forge
        DB_PASSWORD=<het gegenereerde wachtwoord>
  [ ] Fill API keys in Environment:
        POLYGON_API_KEY=...     (historische daily bars, onbeperkt/dag)
        FINNHUB_API_KEY=...     (primaire verse slotkoers, geen daglimiet)
        ALPHA_VANTAGE_API_KEY=... (fallback quote, max 25/dag gratis)
  [ ] Fill overige keys: mail (Resend), VESTIX_ADMIN_NOTIFICATION_EMAILS, PHP_BINARY
  [ ] Resend: domein vestix.io verifiëren, RESEND_API_KEY in Environment
  [ ] Queue worker in Forge (Site → Queue) voor e-mailnotificaties

Deployment (Forge → Site → Deployment)
  [ ] Zero-downtime deploy script: zie forge/deploy.sh
      (Forge variabelen: $CREATE_RELEASE, $FORGE_RELEASE_DIRECTORY, $ACTIVATE_RELEASE)
  [ ] Run first deploy (Deploy Now)
  [ ] SSH-pad naar actieve release: /home/forge/vestix.io/current

Data migreren vanuit lokaal (optioneel)
  [ ] Zelfde APP_KEY als lokaal in Forge Environment (vereist voor api_credentials)
  [ ] Nog GEEN vestix:create-user draaien — users komen uit import

  Lokaal:
  [ ] php artisan vestix:export-data
  [ ] tar -czf vestix-export.tar.gz storage/app/vestix-export/
  [ ] tar -czf vestix-storage.tar.gz storage/app/public/users storage/app/public/position-charts storage/app/public/ticker-logos
  [ ] scp vestix-export.tar.gz vestix-storage.tar.gz forge@vestix.io:/home/forge/vestix.io/

  Op productie (SSH, na eerste deploy + migrations):
  [ ] cd /home/forge/vestix.io/current
  [ ] tar -xzf vestix-export.tar.gz
  [ ] tar -xzf vestix-storage.tar.gz
  [ ] php artisan vestix:import-data storage/app/vestix-export/<timestamp>
  [ ] php artisan storage:link --force
  [ ] Login met bestaand lokaal account — skip vestix:create-user

Scheduler (Forge → Site → Scheduler)
  [ ] Nieuwe scheduled job toevoegen (of controleren dat deze bestaat):
        Command: php artisan schedule:run
        User: forge
        Frequency: Every minute (* * * * *)
        Path: /home/forge/vestix.io/current
  [ ] Verify via SSH: cd /home/forge/vestix.io/current && php artisan schedule:list
      (vestix:fetch-data moet zichtbaar zijn)

First admin (SSH) — alleen bij schone installatie ZONDER data-import
  [ ] php artisan vestix:create-user you@example.com --name="You" --password="..." --super-admin
  [ ] Or register at /admin/register (geen super-admin; gebruik --super-admin voor platformbeheer)

Post-deploy (SSH)
  [ ] php artisan vestix:smoke-test
  [ ] php artisan vestix:telegram-set-webhook
  [ ] Gebruikers: Profiel → Koppel Telegram → Start in bot

Telegram (eenmalig op productie)
  [ ] TELEGRAM_BOT_TOKEN in Forge Environment
  [ ] TELEGRAM_WEBHOOK_SECRET=<random string, bijv. php -r "echo bin2hex(random_bytes(16));">
  [ ] TELEGRAM_BOT_USERNAME=<optioneel, anders auto via getMe>
  [ ] php artisan vestix:telegram-set-webhook
  [ ] Verwijder oude TELEGRAM_CHAT_ID uit .env (niet meer gebruikt)
